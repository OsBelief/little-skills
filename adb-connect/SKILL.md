# ADB 连接 Android TV 配置工具

这个 skill 帮助你通过 ADB 连接到 Android TV 设备，并根据设备型号自动执行相应的调试配置。

## 何时使用

- 用户需要通过 ADB 连接到 Android TV 设备进行调试
- 需要根据设备型号设置不同的调试参数
- 需要配置 Android TV 盒子的调试端口和属性
- 需要获取 Android TV 设备的 root 权限
- 需要验证设备是否已获取 root 权限
- 用户提到"连接电视"、"adb 连接盒子"、"配置 TV 设备"、"获取 root 权限"、"验证 root"等类似表述

### 命令行调用

**通过斜杠命令快速调用：**
```
/adb-connect <IP:PORT>
```

**示例：**
```
/adb-connect 172.33.50.2:5353
/adb-connect 192.168.1.100:5555
```

如果只提供 IP 地址（不指定端口），默认使用 5555 端口：
```
/adb-connect 192.168.1.100
```

## 工作流程

### 步骤 1：确定连接参数

**当通过 `/adb-connect <IP:PORT>` 命令调用时：**
- 从命令参数中提取 IP 地址和端口
- 格式可以是：
  - `IP:PORT`（例如：`172.33.50.2:5353`）
  - 只有 `IP`（例如：`192.168.1.100`），此时使用默认端口 5555

**当通过普通对话调用时：**
- 检查用户是否在消息中提供了 IP 地址和端口
- 如果用户提供了 IP 和端口（格式如 `192.168.1.100:5555`），使用用户提供的值
- 如果用户没有提供，询问用户目标设备的 IP 地址和端口

示例：
```
请提供 Android TV 设备的 IP 地址和端口（例如：192.168.1.100:5555）
```

### 步骤 2：连接到设备

执行 ADB 连接命令：

```bash
adb connect <IP>:<PORT>
```

**错误处理：**
- 如果连接失败，检查 `adb` 是否已安装
- 显示错误信息并停止执行
- 建议用户检查：
  - 设备是否开机并连接到网络
  - ADB 调试是否在设备上启用
  - IP 地址和端口是否正确

**成功输出示例：**
```
✓ 已连接到 <IP>:<PORT>
```

### 步骤 3：获取设备序列号

使用以下命令获取设备型号信息：

```bash
adb -s <IP>:<PORT> shell getprop ro.product.serials
```

**错误处理：**
- 如果命令失败，显示错误信息并停止
- 可能的原因：设备未正确连接、设备不支持此属性、权限不足

**成功输出示例：**
```
✓ 设备序列号: AE1000
```

### 步骤 4：根据设备型号执行配置

根据获取的序列号，执行对应的配置命令：

#### 如果序列号是 AE1000 或 ME60：

```bash
# 设置调试属性
adb -s <IP>:<PORT> shell setprop debug.xy 99

# 设置 TCP/IP 端口
adb -s <IP>:<PORT> tcpip 5555
```

**输出示例：**
```
✓ 检测到设备型号: AE1000
✓ 已设置调试属性: debug.xy = 99
✓ 已配置 TCP/IP 端口: 5555
```

#### 如果序列号是其他型号：

```bash
# 设置调试属性
adb -s <IP>:<PORT> shell setprop debug.link 998

# 设置 TCP/IP 端口
adb -s <IP>:<PORT> tcpip 5555
```

**输出示例：**
```
✓ 检测到设备型号: <序列号>
✓ 已设置调试属性: debug.link = 998
✓ 已配置 TCP/IP 端口: 5555
```

### 步骤 5：获取 root 权限并验证

执行 root 命令并验证是否成功获取权限：

```bash
# 请求 root 权限
adb -s <IP>:<PORT> root

# 等待几秒让设备重启 adbd
# 然后验证 root 权限
adb -s <IP>:<PORT> shell whoami
```

**验证逻辑：**
- 如果 `whoami` 返回 `root`，说明成功获取 root 权限
- 如果返回 `shell` 或其他值，说明 root 权限未获取成功

**成功输出示例：**
```
✓ 正在请求 root 权限...
  adbd is already running as root
  或
  restarting adbd as root

✓ 验证 root 权限...
  当前用户: root
  ✓ 已成功获取 root 权限
```

**失败输出示例：**
```
✗ 正在请求 root 权限...
  adbd cannot run as root in production builds

✗ 验证 root 权限失败...
  当前用户: shell
  ✗ 未获取到 root 权限

建议：
  1. 设备可能需要解锁 bootloader
  2. 设备可能需要刷入具有 root 权限的 ROM
  3. 某些设备不支持 adbd root
```

**错误处理：**
- 如果 `adb root` 失败，显示错误信息并继续执行验证步骤
- 如果验证失败，明确告知用户 root 权限未获取，但不影响之前的配置步骤
- 提供可能的原因和建议

## 输出格式

向用户展示详细的执行步骤和结果，包括：

1. **连接信息**：目标 IP 和端口
2. **设备信息**：检测到的设备型号/序列号
3. **配置步骤**：每个命令的执行状态
   - ✓ 表示成功
   - ✗ 表示失败
4. **Root 权限验证**：是否成功获取 root 权限
5. **命令输出**：显示关键命令的返回值
6. **最终状态**：总结配置是否成功

## 错误处理原则

**遇到任何错误时：**
1. **连接失败**：立即停止执行后续步骤，显示错误信息和建议
2. **获取序列号失败**：立即停止执行，无法确定设备型号
3. **配置失败**：立即停止执行，显示具体哪个命令失败
4. **Root 权限获取失败**：继续执行验证步骤，但明确告知用户权限未获取
5. 不要尝试自动修复或重试（除非明确要求）
6. 保留已完成的步骤信息，让用户了解进度

**特别说明：**
- Root 权限获取失败不应该影响之前的配置步骤
- 即使没有 root 权限，调试参数的配置仍然有效
- 明确告知用户哪些步骤成功，哪些步骤失败

## 常见设备型号

根据已知的设备信息，以下型号会使用特定配置：

- **AE1000**：使用 `debug.xy` 属性，端口 5555
- **ME60**：使用 `debug.xy` 属性，端口 5555
- **其他型号**：使用 `debug.link` 属性，端口 5555

## 注意事项

1. 确保 ADB 工具已安装并在系统 PATH 中
2. 设备必须启用 ADB 调试模式
3. 设备和执行 ADB 的主机需要在同一网络
4. 某些设备可能需要授权 ADB 连接
5. TCP/IP 端口配置后，后续连接可以使用新端口
6. **Root 权限**：
   - 不是所有设备都支持 `adb root`
   - 生产版本的设备通常不允许 adbd 以 root 身份运行
   - 某些设备需要解锁 bootloader 或刷入特定 ROM 才能获取 root 权限
   - 即使无法获取 root 权限，调试参数配置仍然有效
7. 执行 `adb root` 后，adbd 会重启，可能需要等待几秒钟才能重新连接
